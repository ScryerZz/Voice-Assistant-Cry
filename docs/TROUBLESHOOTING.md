# Решение Проблем

Используйте этот файл, если ассистент не устанавливается, не запускается, не слушает, не выполняет команды или неправильно отображается.

## Первая Проверка

```powershell
.\.venv\Scripts\python.exe diagnose.py
```

Если `.venv` отсутствует:

```bat
install.bat
```

## Python Не Найден

Симптомы:

```text
python.exe: system cannot find the path specified
py -0p: No installed Pythons found
```

Целевая версия проекта:

```text
Python 3.11
```

Проверьте:

```powershell
py -0p
where.exe python
```

Если `where.exe python` показывает `WindowsApps\python.exe` раньше реального Python, отключите Windows App Execution Alias для Python или переместите настоящий путь Python выше в PATH.

## Пересоздать Виртуальное Окружение

```powershell
Remove-Item -Recurse -Force .venv
py -3.11 -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe diagnose.py
```

## Ошибка `audioop-lts`

`audioop-lts` нужен только для новых версий Python. В проекте он подключён через marker:

```text
audioop-lts==0.2.2; python_version >= "3.13"
```

На Python 3.11 он устанавливаться не должен.

## UI Не Открывается Или Открывается Слишком Маленьким

Запустите:

```powershell
.\.venv\Scripts\python.exe ui.py
```

Проверьте синтаксис UI:

```powershell
.\.venv\Scripts\python.exe -m compileall ui.py src\ui\config_app.py
```

Главное окно, мастер первого запуска и живой журнал используют адаптивный стартовый размер. Если окно всё равно слишком маленькое, проверьте масштабирование Windows и разрешение экрана.

## Голосовой Режим Не Запускается

Если первичная настройка не завершена, `main.py` открывает UI и отменяет запуск микрофона.

Откройте:

```bat
run_settings.bat
```

Завершите мастер первого запуска.

## Микрофон Или Распознавание Не Работают

Проверьте:

- разрешение Windows на доступ к микрофону;
- микрофон по умолчанию;
- не занят ли микрофон другой программой;
- Vosk-модели;
- настройки `recognition` в `data/config.yaml`.

Если распознавание часто падает, перезапустите ассистента и начните с короткой тестовой фразы.

## Нет Vosk-Модели

Офлайн-распознавание требует модели в `paths.stt_models`, например:

```text
data/models/stt/vosk-model-small-ru-0.22
data/models/stt/vosk-model-small-en-us-0.15
```

Legacy-папки прямо в `data/models/` пока распознаются для совместимости.

## Предупреждение Silero

Пример:

```text
Silero models: warnings: missing or empty ru=data\models\tts\v3_1_ru.pt
```

Это не блокер запуска. Если модель отсутствует, ассистент переключится на `pyttsx3` или текстовый вывод.

## Команда Распознана, Но Не Выполняется

Проверьте:

```powershell
.\.venv\Scripts\python.exe diagnose.py
```

В UI откройте `Команды`, выберите `Проверить фразу` и посмотрите совпадение без выполнения действия.

Если нужные команды не находятся, аккуратно снижайте `matcher.threshold`. Если выполняются неправильные команды, повышайте пороги.

## Отсутствует Action

Каждое действие в `data/commands.yaml` должно соответствовать функции:

```text
src/skills/<module>.py -> <function>()
```

Пример:

```text
rutube.search_rutube -> src/skills/rutube.py -> search_rutube()
```

## ЯндексGPT Не Работает

Проверьте в UI:

```text
Интеграции -> ИИ / ЯндексGPT
```

Нужно заполнить:

- `Отвечать через ИИ на неизвестные команды`;
- `API-ключ ЯндексGPT`;
- `ID каталога Яндекс Облака`.

Важно: ID каталога должен совпадать с каталогом сервисного аккаунта, которому принадлежит API-ключ. Если они не совпадают, ассистент покажет ошибку про ID каталога.

После сохранения нажмите `Обновить контекст` в чате или перезапустите UI.

## Погода Или Новости Не Работают

Проверьте ключи:

```text
Интеграции -> Погода и новости
```

Пустой ключ означает, что функция не настроена. Для проверки используйте команды:

```text
какая погода
последние новости
```

## Приложения Не Открываются

В разделе `Приложения` проверьте:

- путь к `.exe` или ярлыку;
- имя процесса;
- статус найденного пути.

Команда `Подставить найденные пути` заполняет часть популярных приложений автоматически.

## История Показывает Русские Фильтры

UI показывает `Голос`, `Чат`, `Выполнено`, `Ответ ИИ`. В базе остаются стабильные значения `voice`, `chat`, `matched`, `ai_fallback`. Это нормально.

## Отчёт Поддержки Не Экспортируется

Откройте:

```text
Диагностика -> Экспортировать отчёт для поддержки
```

Если экспорт не работает:

- выберите папку, куда есть права записи;
- запустите `diagnose.py` отдельно;
- оставьте `Скрывать секреты при экспорте` включённым;
- временно отключите включение журналов в отчёт, если файл слишком большой.

## Crash Summary Пустой

Crash summary ищет свежие строки с ошибками и предупреждениями в:

```text
data/logs/assistant.log
data/logs/assistant_stdout.log
```

Пустой результат означает, что в выбранном диапазоне таких строк не найдено.

## Профили Не Применяются

Профили меняют только персонализацию: имя, язык, голос, wake words, режим работы и приватность. API-ключи и пути приложений они не трогают.

Если ассистент уже запущен, после применения профиля перезапустите голосовой процесс.

## Логи

Основной журнал:

```text
data/logs/assistant.log
```

Вывод процесса из UI:

```text
data/logs/assistant_stdout.log
```

Команды:

```text
где журнал
последние ошибки
очисти журнал
```

Очистка журнала считается опасным действием и может требовать подтверждения.

## Финальная Сборка `.exe`

Финальная сборка `.exe` ещё не выполняется. Это отдельный этап после явной команды пользователя.
