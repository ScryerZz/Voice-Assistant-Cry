# Дорожная Карта

Этот документ фиксирует актуальный путь Voice Assistant Cry к полноценному продукту и финальной сборке `.exe`.

Финальную сборку не выполнять, пока пользователь явно не даст команду.

## Цель Продукта

Сделать Windows-ассистента, которым обычный пользователь может пользоваться через UI: настроить, запустить, проверить, исправить типовые проблемы, управлять командами и не открывать исходный код.

## Текущее Состояние

Закрыто:

- desktop UI с адаптивным стартовым размером;
- мастер первого запуска;
- чат и голосовые команды через общий `Executor`;
- переключатель голосового считывания;
- история, заметки, напоминания в SQLite;
- профили ассистента;
- интеграции ЯндексGPT, погоды, новостей;
- Rutube-поиск вместо YouTube;
- подтверждение опасных команд;
- диагностика, health snapshot, crash summary, support-отчёт;
- маскировка секретов в отчётах;
- пользовательская документация на русском;
- тесты и `diagnose.py`.

Остаётся до финальной упаковки:

- визуально проверить UI на рабочих разрешениях Windows;
- обновить скриншоты документации;
- выбрать упаковщик;
- собрать `.exe`;
- проверить `.exe` на чистой машине;
- оформить финальные release notes.

## P3 - Финальная Сборка `.exe`

Статус: следующий этап после явной команды пользователя.

### P3.1 Выбор Упаковщика

Нужно решить:

- PyInstaller, Nuitka, MSIX или другой вариант;
- где хранить `data/config.yaml` у установленного приложения;
- включать ли модели в сборку или скачивать отдельно;
- как обновлять приложение;
- что делать с логами, SQLite и пользовательскими командами при обновлении.

### P3.2 Предсборочная Проверка

Обязательные команды:

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests
.\.venv\Scripts\python.exe diagnose.py
.\.venv\Scripts\python.exe -m compileall main.py ui.py diagnose.py src tests
```

Ручная проверка:

- `run_settings.bat`;
- главное окно без ручного растягивания;
- мастер первого запуска;
- живой журнал;
- чат и ЯндексGPT;
- голосовое считывание;
- support-отчёт;
- backup/restore настроек.

### P3.3 Сборка И Проверка `.exe`

После выбора упаковщика:

1. Собрать тестовый `.exe`.
2. Запустить на текущей машине.
3. Запустить на чистой Windows-среде.
4. Проверить UI, чат, голос, диагностику, конфиг, SQLite, логи.
5. Подготовить release notes.
